Cerneala UI

Modern WPF instincts, rebuilt for the MonoGame game loop.

Cerneala brings retained layout, typed properties, templated controls, routed input, and real theming into a real-time frame loop. Idle scheduler checks stay allocation-free, visual order is shared, and only pending work is sorted. No XAML. Not a drop-in replacement.

Framework surface

Built like a UI framework. Tuned for the game loop.

Cerneala isn't a widget kit bolted onto a SpriteBatch. It's a retained tree, a property system, and a render pipeline — the same load-bearing parts WPF developers already know how to reason about.

Tree

Retained UI Tree

The tree persists across frames. Elements mutate in place instead of being rebuilt every draw call, so your UI cost scales with what changed — not with frame count.

Prop

Typed State

UiProperty<T> gives UI state a typed identity, defaults, metadata, and invalidation behavior — the same instincts as WPF dependency properties, minus the XAML.

Ctrl

Familiar Controls

Button, TextBox, ListBox, ScrollViewer, ComboBox, TabControl, Slider — the standard set, templated and stylable from the ground up, not hard-coded skins.

Skin

Theming & Aspect Tokens

Swap palettes, spacing scales, and per-control aspects at runtime — theming is a token layer, not a find-and-replace on style resources.

Move

Motion System

Tweens, springs, keyframes, layout transitions, and presence animations evaluate on the same clock as your game loop — not a separate UI thread pretending to be one.

Route

Routed Input

Events bubble and tunnel through the tree. Commands, focus, and hit-testing follow the same routing model WPF developers already reason about.

Cache

Render Caching

Invalidations deduplicate by element, reuse visual order per tree version, and sort only pending work before dirty render caches refresh.

Diag

Diagnostics

Frame stats, runtime snapshots, invalidation traces, routed event routes, render cache dumps, motion traces, and platform probes make the frame visible instead of guessed at.

Draw

Backend-Neutral Pipeline

UI emits an abstract drawing command stream. The included MonoGame adapter consumes it today — the pipeline itself doesn't assume MonoGame.

Frame pipeline

One frame, retained stages.

Cerneala moves input into pixels through a retained scheduler: input can dirty the tree, queued properties and aspects settle first, measure and arrange update layout, render caches refresh only where needed, and the backend submits the committed command stream.

IN
Input

Pointer, keyboard, and text input dispatch through hit testing, capture, focus, commands, navigation, and routed events.

AS
Props & Aspects

UiProperty<T>, inherited values, command state, and aspect rules settle before layout work is taken.

ME
Measure

Dirty measure work snapshots only pending entries against a shared visual-order index, without rebuilding order per queue.

AR
Arrange + Motion

Arrange writes final bounds while motion hooks sample around layout and render for transitions, presence, and reduced-motion policy.

RC
Render Cache

Render invalidations refresh element caches, rebuild root commands when stale, then update hit-test routes for the next input pass.

BE
Backend

The host submits the committed drawing commands to an IDrawingBackend; the included adapter renders them through MonoGame.

Runtime diagnostics

See inside the frame.

Cerneala exposes frame snapshots, invalidation traces, dirty-tree dumps, routed event routes, render cache diagnostics, layout captures, input targets, motion traces, and platform/resource probes.

Use the diagnostic layer to see what changed, which caches were reused, why layout ran, how input routed, and what services the runtime can actually reach.

Cerneala UI

Cerneala takes cues from WPF's retained-mode mental model — trees, typed properties, templated controls — reworked for MonoGame's frame loop. It is not a drop-in WPF replacement, does not support XAML, and does not yet provide full native accessibility support. Production-hardening is ongoing.
