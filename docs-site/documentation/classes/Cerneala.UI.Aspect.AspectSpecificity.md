# AspectSpecificity Struct

## Examples

Create specificity values and combine them:

```csharp
using Cerneala.UI.Aspect;

AspectSpecificity component = new(Component: 1);
AspectSpecificity hoveredVariant = new(Variant: 1, State: 1);

AspectSpecificity combined = component + hoveredVariant;

bool combinedWins = combined.CompareTo(component) > 0;
```

Use condition specificity while building a target:

```csharp
using Cerneala.UI.Aspect;
using Cerneala.UI.Controls;

AspectTarget target = new(
    typeof(Button),
    conditions: [AspectCondition.State(AspectState.Hover)]);

AspectSpecificity specificity = target.Specificity;
```

## Remarks

`AspectSpecificity` is the score carried by aspect conditions and targets. `AspectTarget` starts with component and slot specificity, then adds the specificity of each target condition. Compound conditions add `Compound` specificity plus their children, while state, variant, property, data, and predicate conditions increment their matching category.

`CompareTo` compares fields in this order: `Component`, `Slot`, `Variant`, `State`, `Property`, `Data`, `Compound`, then `Predicate`. The first non-equal field decides the ordering. This ordering is used by `AspectCascadeKey` after layer order and before declaration order.

The `+` operator returns a new `AspectSpecificity` whose fields are the per-category sums of the left and right operands. The type is a readonly record struct, so its public constructor parameters are exposed as init-only value properties and value equality is generated by the compiler.
