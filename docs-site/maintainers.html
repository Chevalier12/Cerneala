PRIME DIRECTIVE Do not raw-dog this repository with your unaided human brain.

Cerneala has the surface area of a repository that would normally require 40+ developers. Its maintainer is one person. Trying to understand every subsystem at once is not diligence; it is a fast route to losing your mind. Give a capable agent the exact scenario, let it load the local context, read the repository instructions, navigate through RoslynIndexer, run focused experiments, create tests, and show you evidence. Your job is intent, judgment, review, and deeply understanding the slice you are actually changing.

BUG PROTOCOL

Reproduce. Observe. Fail red. Fix. Reproduce again.

Do not begin with a theory about the implementation. Begin with the user-visible failure and enough telemetry for GPT-5.6 to find the actual contract violation.

  1. 01
    Discover and reproduce

    Reduce the bug to the smallest reliable sequence. Record the custom view, exact actions, expected behavior, actual behavior, frequency, and environment.

    Bonus points: include the frame-stats TextBlock used by the Playground.
  2. 02
    Give GPT-5.6 the scenario

    Tell it where the bug appears, what your custom view contains, what you did, what happened, and precisely what pisses you off about it.

  3. 03
    Reproduce with CSI first

    Require a focused temporary .csx experiment. The agent should inspect all relevant state transitions, use a short timeout, clean up the script, and kill any suspicious stuck CSI process.

  4. 04
    Create a RED test

    Encode the failure before modifying production behavior. The test must fail for the observed reason, not because the test harness is broken.

  5. 05
    Implement the smallest correct fix

    Respect existing ownership and architecture boundaries. Fix the violated contract without unrelated refactors, speculative abstractions, or compatibility theater.

  6. 06
    Retest with CSI

    Run the same focused experiment again. Confirm the original reproduction is gone and inspect the state that previously exposed the bug.

  7. 07
    Test the real scenario manually

    Open the affected view or Playground scenario and perform the original interaction yourself. Inspect the visual result, input, layout, diagnostics, and frame stats for anything obviously fucked.

  8. 08
    Run the full test suite until GREEN

    The model always runs the entire suite, not merely the focused tests. If your change broke another contract, it must tell you exactly what regressed. Tell it to fix the regression and rerun everything.

    A green full suite is mandatory. No exceptions.
  9. 09
    Commit, push, open the PR

    Include the reproduction, red-test evidence, implementation summary, CSI retest, manual test, and full green-suite result. The maintainer reviews from there.

IMPLEMENTATION PROTOCOL

Discuss intent. Freeze a staged plan. Execute the checklist.

Large implementation work begins as a conversation, not a blind code dump. Once intent and boundaries are clear, the plan becomes persistent project memory.

  1. 01
    Discuss the implementation

    Explain the scenario, desired API, developer experience, constraints, non-goals, integration points, and what success looks like. Let the agent challenge unclear assumptions.

  2. 02
    Create a new staged plan file

    Require a dedicated plan under docs/plans/. It must be split into coherent phases and use checklist items that are checked as work is completed.

  3. 03
    Implement phase by phase

    Follow repository patterns, keep scope tight, reindex after code changes, test each contract, update API docs with public surface changes, and mark the plan honestly.

  4. 04
    Manually test the complete scenario

    Use the real Playground or affected application path. Exercise the implementation like a developer would, inspect visuals and interaction, and check diagnostics and frame stats for collateral damage.

  5. 05
    Run the full test suite until GREEN

    The agent always runs every repository test. If your implementation broke existing behavior, it reports the regression. Tell it to fix the damage and run the complete suite again.

    A green full suite is mandatory. No exceptions.
  6. 06
    Commit, push, open the PR

    Link the plan, summarize architecture and public behavior, list tests and manual validation, include the full green-suite result, call out limitations, and hand the focused change to the maintainer.
